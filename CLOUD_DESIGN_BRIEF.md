# Clinky — Website Build Brief (for Cloud Design)

> Задача для AI-агента дизайна («Cloud Design»): собрать **максимально продающий pre-launch waitlist-лендинг** для iOS-приложения **Clinky**, разместить его в этом Git-репозитории (GitHub Pages). Всё необходимое — ниже. Тексты, бренд-токены, ассеты и ограничения уже подготовлены. Не выдумывай факты — используй этот документ как источник истины.

---

## 0. TL;DR
- **Что строим:** одностраничный лендинг-«очередь» (waitlist). Цель — заманить, показать фишки (особенно игры) и собрать email, чтобы на релизе разослать ссылку на App Store.
- **Стек:** статический сайт для GitHub Pages — **чистый HTML/CSS/JS без сборки** (или фреймворк, который билдится в статику). Бэкенда нет.
- **Языки:** RU + EN, переключатель с запоминанием выбора, автоопределение из браузера.
- **Стиль:** яркий малиновый/коралловый bold-герой + светлые секции; шрифт Nunito (заголовки) + DM Sans (текст). Энергично, playful, party/social.
- **Фишка:** живое 3D (вращаемая пробка пива / стакан кофе) + интерактивная карточка игр с реальными вопросами.
- **Уже в репо:** легальные страницы, 360°-кадры моделей, USDZ для AR, JSON вопросов, рабочий baseline `index.html`. **Не ломай их.**

## 1. Git и деплой — как работать с этим репозиторием

- **Repo:** `https://github.com/EvgeniyMityulya/clinky-legal` (public).
- **Клонировать:** `git clone https://github.com/EvgeniyMityulya/clinky-legal.git`
- **Хостинг:** GitHub Pages раздаёт **ветку `main`, корень `/`** по адресу
  **`https://evgeniymityulya.github.io/clinky-legal/`**. Любой push в `main` авто-деплоится за ~1 минуту. Файл `.nojekyll` в корне обязателен (отключает Jekyll) — не удаляй.
- **Рабочий цикл:**
  1. `git pull` → правки → `git add -A`
  2. `git commit -m "feat: <что сделал>"` (Conventional Commits, на английском)
  3. `git push origin main` → проверь живую страницу через ~1 мин.
- **Авторизация для push:** нужен доступ к аккаунту `EvgeniyMityulya`. Владелец выдаёт **Personal Access Token (scope: `repo`)** или пушит сам. Без валидного токена — только локальные коммиты, push владелец делает руками.
- **Проверка после деплоя:** открой URL выше; убедись, что `index.html`, `assets/*`, `models/spin/*`, `privacy.html` отдают HTTP 200.

## 3. Что такое Clinky (продукт)
**Clinky** — iOS-приложение (iOS 17+): социальный трекер встреч и напитков + **карточки-игры (ice-breakers)** для живых встреч с друзьями. Offline-first, без аккаунтов, без бэкенда.
- **Tagline:** «Tap. Clink. Meet again.»
- **Суть:** помни людей, с кем видишься; отмечай встречи и напитки; разговаривай компанию карточками-играми; собирай 3D-сувенир («collectible») за каждую встречу; следи за сериями и статистикой.
- **Бренд-термины (НЕ переводить):** Clinky, Clinkers (друзья), Clink, Pro.
- **Статус:** ещё НЕ опубликовано в App Store → CTA = **waitlist** («Скоро» / «Встать в очередь»), не «Скачать».

## 4. Аудитория и позиционирование
- Молодая, social, любит компании, вечеринки, настолки/вопросы-игры.
- Позиционирование = пересечение трёх ниш: **друзья-трекер + любой напиток (пиво/кофе/что угодно) + party/ice-breaker игры**. **Игры — сильнейший крючок** (и для конверсии, и для трафика).
- **Не «алкогольное» приложение:** Clinky про встречи. Любой напиток — даже **без алкоголя** (кофе). Рейтинг 17+ (есть упоминания алкоголя), но фрейминг — про общение, не про выпивку.

## 5. Пять вкладок приложения (детально)
Это «выборка вкладок» — ключевой интерактив сайта (телефон-макет + таб-бар, тап по табу меняет экран и текст).
1. **Home / Главная** 🏠 — ближайшая запланированная встреча, активная Live-сессия, лента недавних встреч, серии.
2. **Clinkers / Друзья** 👥 — список всех друзей, у кого давно не виделись видно сразу, свайп-удаление, карточка друга с фактами.
3. **Live (центр) / Live** 🍻 — старт живой встречи: **ice-breaker карточки с вопросами**, отметка напитков, по итогу — 3D-collectible. Это «сердце» приложения.
4. **Analytics / Аналитика** 📊 — графики: кого видишь чаще, серии, любимые напитки, факты.
5. **Profile / Профиль** 👤 — достижения, выбор иконки приложения, Clinky Pro, настройки.

## 6. Четыре игры (с реальными вопросами)
Полные вопросы — в `assets/icebreakers.json` (поле `text.en` / `text.ru`). Игры:
1. **Never have I ever / Я никогда не** 🙈 — напр.: *«Я никогда не придумывал целые диалоги в голове с теми, кого только собирался встретить.»*
2. **Roulette / Рулетка** 🎯 — вопрос про двух участников (в JSON плейсхолдеры заменены примерными именами Аня/Макс): *«Аня, что в Максе тебя восхищает больше всего?»*
3. **Tell a moment / Расскажи момент** 💬 — напр.: *«Расскажи самый странный сон, который тебе снился.»*
4. **Would you rather / Что выберешь** ⚖️ — напр.: *«Что выберешь: знать ответ на любой вопрос или уметь убедить любого в чём угодно?»*

## 7. 3D-коллекционные напитки (ассеты готовы)
- В приложении 5 моделей; для САЙТА используем **только Пиво и Кофе** (алкоголь / без алкоголя — сильный месседж инклюзивности).
- **Готовые ассеты в `models/spin/`:** по 36 кадров на оборот 360°, 720px, прозрачный фон: `beer_00.png … beer_35.png`, `coffee_00.png … coffee_35.png`.
  - **Кофе** = красный стакан с крафт-держателем и белой крышкой (в бренде, цветной).
  - **Пиво** = серебристая металлик-пробка (чистый минималистичный объект).
- **Реализация спиннера (рекомендация):** прелоад кадров, drag/touch вращает (X → индекс кадра), авто-вращение в простое; переключатель 🍺/☕ меняет набор кадров. Уважай `prefers-reduced-motion` (без авто-вращения).
- **AR на iPhone:** `<a rel="ar" href="models/BeerCap.usdz">` (и `CoffeeCup.usdz`) — открывает AR Quick Look.

## 12. Интерактив — требования
- **i18n RU/EN:** словарь в JS, атрибуты `data-i18n`/`data-i18n-attr`, `localStorage`, автодетект `navigator.language`. Переключатель меняет `<html lang>`, тексты, ссылки легалки (по локали), контент карточки игр и подписи вкладок.
- **3D-спиннер:** из кадров `models/spin/`. Минимальный приём:
  ```js
  const frames = Array.from({length:36}, (_,i)=>`models/spin/${drink}_${String(i).padStart(2,'0')}.png`);
  // preload → на drag: idx = Math.floor((pointerX/width)*36) % 36; img.src = frames[idx];
  // idle: автоинкремент idx раз в ~80мс (если не reduced-motion)
  ```
- **Карточка игр:** `fetch('assets/icebreakers.json')` → табы по `games[].title[lang]` → показывай `questions[i][lang]`, «Следующий вопрос» листает, плавный fade.
- **Витрина вкладок:** 5 кнопок таб-бара → меняют экран (эмодзи/плейсхолдер) + заголовок/описание (см. копи).
- **Форма waitlist:** валидация email; пока **бэкенда нет** — на сабмит показывай success-состояние (`hero.done`). Точку интеграции оставь явной: `form.action`/`data-endpoint` под будущий сервис (MailerLite / Kit / Brevo). Не имитируй реальную отправку.
